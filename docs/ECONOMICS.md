# Economics Notes

## Product Economics

StackSave is designed as a low-complexity SaaS product with lightweight
infrastructure costs and a clear value proposition: help teams find savings
that exceed the cost of the audit itself.

## Cost Drivers

- Vercel hosting for the web app
- Supabase for persistence and relational data
- Resend for transactional email delivery
- Anthropic API usage for summary generation
- maintenance time for pricing rules and documentation

## Value Drivers

- seat consolidation
- plan right-sizing
- identifying unused subscriptions
- surfacing annualized savings opportunities

## Economic Assumption

The product is strongest when it creates a savings estimate that is large
enough to justify follow-up, implementation work, or an internal budget review.
In other words, the audit should produce enough measurable value to make the
next action obvious.

## Monetization Possibilities

- one-time audit fees
- freemium audit with paid follow-up services
- subscription access for historical reporting and team tracking
- bundled consulting for implementation support

## Risk Notes

- overly aggressive savings claims can reduce trust
- inaccurate pricing assumptions can weaken recommendation quality
- AI usage costs should stay bounded because the core product value is not
  dependent on large model volumes

## Strategic Principle

Keep the product economically simple: the more the savings logic stays
deterministic and reusable, the easier it is to explain margins, pricing, and
customer value.
