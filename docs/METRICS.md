# Metrics Notes

## Product Metrics

These metrics help evaluate whether StackSave is producing useful audits and
converting interest into leads.

## Acquisition Metrics

- landing page conversion rate
- audit form start rate
- audit form completion rate
- results page view rate

## Value Metrics

- average monthly savings identified
- average annual savings identified
- percentage of audits with at least one recommendation
- average number of recommendations per audit

## Funnel Metrics

- public report share rate
- lead form completion rate
- duplicate lead rate
- email deliverability rate

## Quality Metrics

- validation error rate on the audit form
- API error rate for audit and lead submission
- report load success rate
- deterministic test pass rate for audit calculations

## Suggested Reporting Cadence

- weekly for funnel and error monitoring
- monthly for savings trends and lead conversion
- per release for test and reliability checks

## Metric Principle

Prefer metrics that connect product usage to credible savings and follow-up
interest. Vanity metrics alone do not show whether the audit is helping users
make a decision.
