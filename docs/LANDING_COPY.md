# Landing Copy Notes

## Brand Voice

StackSave should sound clear, practical, and financially credible. The copy
should focus on savings, clarity, and fast decision-making rather than hype.

## Homepage Messaging

Suggested homepage structure:

1. headline that frames AI spend as a hidden budget problem
2. short supporting line about finding unused seats and overlapping tools
3. primary CTA for starting the audit
4. secondary CTA for seeing an example result

## Value Statements

- find wasted AI spend in minutes
- quantify monthly and annual savings
- surface the best next action for each team
- share a clean report with stakeholders

## Proof Points

Copy should reinforce that the product is:

- deterministic where finance matters
- easy to use without setup
- designed for startup teams
- built to produce a report that can be forwarded internally

## CTA Guidance

- keep the primary CTA action-oriented
- use savings language instead of generic product language
- avoid promising full procurement automation
- emphasize audit, report, and savings outcomes

## Example Direction

The landing page should answer the visitor’s core question immediately:

What AI spend can we remove without hurting productivity?

That framing fits the current MVP and keeps the page aligned with the audit
flow.
